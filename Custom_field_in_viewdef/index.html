Custom field trong EditViewDef

Chú ý: Những phần nào thiếu thì tự tạo thêm.
  • Vị trí: custom/Module/<module>/metadata/
  • Khai báo field cần custom trong phần panels của viewdef:
<?php
array(
    'name' => '<field_name>',
    'label' => '<label_field>',
    'customCode' => '{file="custom/modules/<module>/tpls/<file_name>.tpl"}'
);
  • Ví dụ: custom field ticker_symbol trong module Account
  • Vào file editviewdefs.php chỉnh sửa field ticker_symbol
<?php
array(
     'name' => 'ticker_symbol',
     'label' => 'LBL_TICKER_SYMBOL',
     'customCode' => '{include file="custom/modules/Accounts/tpls/customTicker.tpl"}'
),
  • Trong file custom/modules/Accounts/tpls/customTicker.tpl
<select name="ticker_symbol">
   <option  value="A">A</option>
   <option  value="B">B</option>
   <option  value="C">C</option>
   <option  value="D">D</option>
</select>
  • Sau khi thực hiện các bước trên ta nhận thấy field ticker_symbol lưu được kết quả nhưng không hiển thị giá trị khi edit. Nguyên nhân do field custom nên hệ thống không hiển thị được giá trị đã chọn.
  • Hướng xử lý: truyền dữ liệu vào file tpl để hiển thị.
  • Override file view.edit.php để truyền dữ liệu vào tpl file
    • Vị trí: custom/Module/<module>/views/view.edit.php
<?php
if(!defined('sugarEntry') || !sugarEntry) die('Not A Valid Entry Point');
class AccountsViewEdit extends ViewEdit
{
 public function display()
 {
     //truyền giá trị của field ticker_symbol vào tpl      
     $this->ss->assign("ticker_symbol", $this->bean->ticker_symbol);
     parent::display(); // TODO: Change the autogenerated stub
 }
}

  - Sửa lại file `customTicker.tpl`

<select name="ticker_symbol">
  <option {if $ticker_symbol == 'A'} selected {/if} value="A">A</option>
  <option {if $ticker_symbol == 'B'} selected {/if} value="B">B</option>
  <option {if $ticker_symbol == 'C'} selected {/if} value="C">C</option>
  <option {if $ticker_symbol == 'D'} selected {/if} value="D">D</option>
</select>
